# V5：复杂聊天机器人 — 需求规格说明书

> **前置条件**：V1–V4 已完成（会话列表、聊天气泡、学习模式、Markdown 渲染、真实大模型流式接入、错误处理与重试机制）。

---

## 1. 总览

### 1.1 目标

将 AI ChatBot 从一个基础聊天助手升级为功能完善的**学习生产力工具**，涵盖会话深度管理、Prompt 模板体系、代码交互、知识收藏与导出、暗黑主题等能力。

### 1.2 功能全景

| 优先级 | 功能模块 | 简述 |
|--------|----------|------|
| P0 | 多会话管理增强 | 重命名、置顶、归档、搜索、拖拽排序、批量删除 |
| P0 | 会话标题自动生成 | 调大模型生成摘要标题，替代截断方式 |
| P0 | 暗黑模式 | 跟随系统 + 手动切换 |
| P1 | Prompt 模板市场 | 内置模板库 + 用户自建 + 搜索分类 |
| P1 | 本地历史搜索 | 全局搜索所有会话的消息内容 |
| P1 | 收藏优秀回答 | 标记/取消收藏消息，收藏列表 |
| P1 | 导出聊天记录 | JSON 全量导出 + Markdown 单会话导出 |
| P2 | 上传代码片段 | 粘贴 / 文件上传代码，格式化展示 |
| P2 | AI 代码解释 | 选中代码一键让 AI 解释 |
| P2 | AI 学习计划生成 | 独立的学习计划生成工作流 |
| — | 远期规划 | 工具调用展示、AI 思考步骤卡片、代码检查结果卡片、待办事项生成、学习进度记录 |

### 1.3 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | Next.js 16 App Router | 沿用 |
| UI 库 | React 19 + Ant Design 6 + Tailwind CSS 4 | 沿用 |
| 状态管理 | 自定义 Hook (`useChatStore`) | 沿用，按需扩展 |
| 持久化 | localStorage | 沿用，模板/收藏等新增独立 key |
| 拖拽 | `@dnd-kit` 或原生 HTML DnD | 会话列表排序 |
| 主题 | CSS Variables + `prefers-color-scheme` | 暗黑模式方案 |
| 导出 | 前端 Blob + `URL.createObjectURL` | 纯前端导出，无需后端 |

### 2.1 多会话管理增强（P0）

#### 2.1.1 现状

当前 `SessionList` 组件支持：新建、切换、删除会话。会话标题由第一条消息前 20 字截断生成。

#### 2.1.2 新增能力

| 能力 | 交互说明 |
|------|----------|
| **重命名** | 双击会话标题进入编辑态，失焦或回车保存 |
| **置顶/取消置顶** | 右键菜单或 hover 时显示📌图标，置顶会话固定在列表顶部 |
| **归档** | 右键菜单「归档」，归档会话从主列表移入「已归档」折叠区 |

| **批量删除** | `Ctrl+Click` 多选会话，或点击顶部「选择」按钮进入多选模式，勾选后可批量删除 |

### 2.2 会话标题自动生成（P0）

#### 2.2.1 现状

`generateTitle()` 截取用户第一条消息前 20 字符作为标题。

#### 2.2.2 升级方案

第一轮对话完成后（用户消息 + AI 回复都已就位），异步调用大模型生成摘要标题：

**后端新增接口**：

```
POST /api/generate-title
```

**请求体**：

```ts
interface GenerateTitleRequest {
  userMessage: string     // 用户第一条消息
  assistantReply: string  // AI 回复（截取前 200 字即可）
}
```

**响应**：

```ts
interface GenerateTitleResponse {
  title: string  // 生成的标题（不超过 20 字）
}
```

**System Prompt**：

```
你是一个标题生成器。根据对话内容生成一个简短、精准的会话标题（不超过15个字），只输出标题文本，不要加引号或其他格式。
```

#### 2.2.3 前端流程

```
第一轮对话完成
  → 异步 POST /api/generate-title（不阻塞 UI）
  → 成功后更新会话标题
  → 失败则保留原来的截断标题（静默降级）
```

- 用户手动重命名过的会话**不再**自动更新标题
- 请求加 `AbortController`，切换页面时取消

### 2.3 暗黑模式（P0）

#### 2.3.1 方案

采用 **CSS Variables + `prefers-color-scheme`** 方案：

```css
/* 亮色主题（默认） */
:root {
  --bg-primary: #ffffff;
  --bg-secondary: #f5f5f5;
  --text-primary: #1a1a1a;
  --text-secondary: #666666;
  --border-color: #e8e8e8;
  --bubble-user: #1677ff;
  --bubble-robot: #ffffff;
  /* ... */
}

/* 暗黑主题 */
[data-theme="dark"] {
  --bg-primary: #1a1a2e;
  --bg-secondary: #16213e;
  --text-primary: #e0e0e0;
  --text-secondary: #a0a0a0;
  --border-color: #2a2a3e;
  --bubble-user: #0f4c81;
  --bubble-robot: #242438;
  /* ... */
}
```

#### 2.3.2 切换逻辑

| 行为 | 说明 |
|------|------|
| 默认 | 跟随系统 `prefers-color-scheme` |
| 手动切换 | 右上角主题按钮（☀️/🌙），存入 `localStorage` key `theme_preference` |
| 优先级 | 手动选择 > 系统设置 |

#### 2.3.3 实现要点

- `chatStyles.css` 中所有硬编码颜色值替换为 CSS Variable
- Ant Design 组件通过 `ConfigProvider` 的 `theme.algorithm` 切换（`defaultAlgorithm` / `darkAlgorithm`）
- 在 `<html>` 标签上设置 `data-theme` 属性
- 首次加载时读取 `localStorage`，若无则读取 `window.matchMedia('(prefers-color-scheme: dark)')`

### 2.5 本地历史搜索（P1）

#### 2.5.1 功能概述

全局搜索所有会话中的消息内容，快速定位历史对话。

#### 2.5.2 UI 设计

- 左侧边栏会话列表顶部增加搜索框
- 支持快捷键 `Ctrl+K`（Windows）/ `Cmd+K`（Mac）唤起搜索
- 输入关键词后实时过滤，按相关性排序

**搜索结果展示**：

```
┌─────────────────────────────────┐
│ 🔍 React 组件          [ESC关闭]│
├─────────────────────────────────┤
│ 📂 React 学习 (会话名)          │
│   ─ "React 的核心是组件..."     │
│   ─ "组件拆分的原则是..."       │
│                                 │
│ 📂 Hooks 笔记                   │
│   ─ "useState 用于组件状态..."  │
└─────────────────────────────────┘
```

- 匹配关键词高亮显示
- 点击搜索结果 → 跳转到对应会话并滚动到该消息位置
- 搜索范围：所有会话的 `message.content`（包括用户和机器人消息）

#### 2.5.3 实现要点

- 纯前端实现，遍历 `sessions` 数组做字符串匹配
- 输入防抖 300ms
- 会话数量较多时（>100），使用 Web Worker 避免主线程卡顿（可选优化）

### 2.7 导出聊天记录（P1）

#### 2.7.1 功能概述

支持将聊天记录导出为 **JSON** 或 **Markdown** 格式，纯前端实现。

#### 2.7.2 导出入口

| 入口 | 导出范围 |
|------|----------|
| 会话列表右键菜单 | 单个会话 |
| 侧边栏底部「⬇ 导出」按钮 | 全部会话 |

#### 2.7.3 JSON 格式

```json
{
  "exportTime": "2024-01-15T10:30:00.000Z",
  "version": "1.0",
  "sessions": [
    {
      "id": "uuid",
      "title": "React 学习",
      "createdAt": 1705312200000,
      "messages": [
        {
          "role": "user",
          "content": "什么是React?",
          "timestamp": 1705312200000
        },
        {
          "role": "assistant",
          "content": "React是...",
          "timestamp": 1705312210000
        }
      ]
    }
  ]
}
```

#### 2.7.4 Markdown 格式

每个会话生成一个 `.md` 文件，内容为：

```markdown
# React 学习

> 创建时间：2024-01-15 10:30

---

**用户**：什么是React?

**AI**：React是一个用于构建用户界面的JavaScript库...

---

**用户**：组件怎么拆分？

**AI**：组件拆分遵循单一职责原则...
```

#### 2.7.5 实现要点

- 使用 `Blob` + `URL.createObjectURL` 触发浏览器下载
- 单会话导出：直接下载
- 全量导出 JSON：单个 `.json` 文件
- 全量导出 Markdown：打包为 `.zip`（引入 `jszip` 库）

### 2.8 上传代码片段（P2）

#### 2.8.1 功能概述

用户可以在输入框中附带代码片段发送给 AI，代码以格式化方式展示。

#### 2.8.2 交互方式

| 方式 | 说明 |
|------|------|
| **粘贴** | 在输入框粘贴代码时自动识别，弹出提示「检测到代码，是否以代码块发送？」 |
| **文件上传** | 输入框左侧增加 📎 附件按钮，支持 `.js` `.ts` `.jsx` `.tsx` `.py` `.html` `.css` `.json` |
| **手动插入** | 输入框工具栏增加 `</>` 按钮，点击弹出代码输入弹层 |

#### 2.8.3 消息展示

用户消息中代码部分以语法高亮代码块渲染（复用现有 `CodeBlock` 组件），文本部分正常展示。

#### 2.8.4 数据模型

```ts
interface ChatMessage {
  id: string
  role: ChatRole
  content: string
  timestamp: number
  error?: ChatErrorMessage
  // ---- V5 新增 ----
  codeAttachment?: {
    language: string    // 语言标识
    code: string        // 代码内容
    filename?: string   // 文件名（上传时有）
  }
}
```

#### 2.8.5 发送逻辑

代码片段作为用户消息的一部分发送给后端：

```
用户的文字描述（如果有的话）

\`\`\`language
code content
\`\`\`
```

### 2.9 AI 代码解释（P2）

#### 2.9.1 功能概述

选中消息中的代码块，一键让 AI 逐行解释代码含义。

#### 2.9.2 交互流程

```
hover 代码块 → 右上角出现「💡 解释代码」按钮
  → 点击后自动发送一条消息：
    "请逐行解释以下代码：\n\`\`\`lang\n[code]\n\`\`\`"
  → AI 以普通对话形式返回解释
```

#### 2.9.3 System Prompt 补充

当触发代码解释时，自动附加额外的指令：

```
请用初学者能理解的语言，逐行解释这段代码的作用。
对每一行或每一小段，说明：
1. 这行代码做了什么
2. 为什么要这样写
3. 有没有更好的写法
```

### 2.10 AI 学习计划生成（P2）

#### 2.10.1 功能概述

提供独立的学习计划生成工作流，区别于普通对话的 plan 模式。

#### 2.10.2 工作流

```
步骤1：用户填写表单
  ├─ 学习目标：[________________]
  ├─ 当前水平：[零基础 ▼] / [初级] / [中级] / [高级]
  ├─ 可用时间：[每天___小时，共___周]
  └─ 特殊需求：[________________]

步骤2：AI 生成计划
  → 调用大模型，传入上述信息
  → 流式输出结构化学习计划

步骤3：计划展示
  → 以卡片形式展示：
    - 总体目标
    - 每周安排（表格形式）
    - 每日任务清单（可勾选）
    - 推荐资源链接
    - 里程碑节点
```

#### 2.10.3 数据存储
